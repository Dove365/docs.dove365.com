# User Guide - Cases

## Purpose

Cases are the main support records in Dove365 Support. They track customer issues, requests, service questions, complaints, billing queries, changes, and general enquiries.

## Create a Case

Cases can be created manually or automatically from support email triage.

Manual steps:

1. Open Cases.
2. Select New.
3. Enter the Case name/title.
4. Select the Account.
5. Select the Contact where known.
6. Select priority, source, type, Category, and Subcategory.
7. Select Service Offering.
8. Link Managed Service if applicable.
9. Enter details and target resolution date.
10. Save.

## Link related records

A good Case should be linked to:

- Account
- Contact
- Category
- Subcategory
- Service Offering
- Managed Service when relevant

Missing links reduce reporting quality and may cause the Case to appear in attention dashboards.

## Business process flow

The Support Case Resolve process guides the Case through resolution.

Users should complete required Case information before moving to resolution.

Typical lifecycle:

1. Intake.
2. Triage.
3. Investigation.
4. Work and communication.
5. Resolution details.
6. Close Case.

## Investigation

During investigation:

- Review the customer history.
- Review Managed Service context.
- Review related activities.
- Add Support Activities for work performed.
- Update status and priority as needed.

## Escalation

Escalate a Case when manager or specialist attention is required.

Before escalating:

1. Confirm the reason for escalation.
2. Set escalation fields.
3. Confirm the escalation manager is populated.
4. Add clear notes.

The `Notify Case Escelation Manager` automation notifies the escalation manager when the Case is escalated.

## Resolution

Before resolving:

1. Confirm the customer issue is complete.
2. Confirm Support Activities and hours are entered.
3. Confirm Managed Service usage is correct.
4. Enter resolution details.
5. Use the Case resolution process to close the Case.

The `Support Case - Resolve` automation closes the Case and notifies the customer.

## Cancel or inactivate

Only cancel or inactivate a Case where it should not be completed as normal support work.

Use clear notes so future users understand why the Case was not resolved.
