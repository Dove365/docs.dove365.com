# User Guide - Managed Services

## Purpose

Managed Services represent contracted or agreed support allocation for an Account and Service Offering.

Use Managed Services to track:

- Allocation hours.
- Hours used.
- Hours remaining.
- Usage percentage.
- Start and end dates.
- Related Cases.

## Who can create Managed Services

Managed Service creation is intended for users with the Dove365 Support Account Manager role or Dove365 Support Admin role.

The Dove365 Support User role can append or reference Managed Services where permitted but is not intended to administer them.

## Creating a Managed Service

Steps:

1. Open Managed Services.
2. Select New.
3. Choose the Account.
4. Choose the Service Offering.
5. Enter the monthly allocation.
6. Enter start and end dates.
7. Save the record.
8. Confirm the record is active.

## Managing Managed Services

Account managers should review Managed Services regularly.

Check:

- Current allocation.
- Hours used.
- Hours remaining.
- Usage percentage.
- End date.
- Related Cases.

## Using Managed Services on Cases

When a Case is covered by a Managed Service:

1. Set the Case as managed-service related where applicable.
2. Select the correct Managed Service.
3. Log Support Activities with time.
4. Confirm hours roll up correctly.

## Hours and usage

Support Activity hours contribute to Managed Service usage through automation and plugin logic.

Interpretation:

- Hours Used: time already consumed.
- Hours Remaining: allocation still available.
- Usage Percentage: how much of the allocation has been consumed.
- 80 percent or above: review soon.
- 100 percent: allocation is fully used.
- Over 100 percent: over-allocation.

## Daily threshold automation

The daily 80 percent threshold automation identifies Managed Services that are approaching allocation.

Users should review these records and decide whether to:

- Continue under the existing agreement.
- Pause work.
- Discuss additional allocation.
- Escalate commercially.

## Best practice

- Link Cases to Managed Services when the work consumes an allocation.
- Log Support Activities promptly.
- Review over-allocation before resolving or continuing high-effort work.
