Documentation/Security Roles

Security Roles Guide

Assign Dove365 Support access based on each person’s operational responsibility.

Role Model

Least privilege for support work

Security roles control what users can see and do in Dove365 Support. Assign the least privileged role that lets each user complete their job, and use teams where you need to manage access at scale.

01

Support User

Day-to-day support agents who pick queue work, update Cases, log Support Activities, and use dashboards.

02

Support Account Manager

Account owners who maintain customer records and Managed Services, and review support allocation usage.

03

Support Admin

Administrators who configure reference data, queues, categories, environment values, and Support app setup.

Role Capabilities

What each role should allow

RoleTypical responsibilities
Dove365 Support UserWork queue items, create and update Cases, log Support Activities, review dashboards, and resolve assigned work.
Dove365 Support Account ManagerMaintain Accounts, Contacts, Managed Services, usage context, and customer support coverage.
Dove365 Support AdminMaintain categories, queues, environment values, mailbox triage settings, roles, and post-deployment validation.
Assignment Process

Assign roles deliberately

✓

Start with the user’s responsibility

Decide whether the user is a support agent, account manager, or administrator before assigning access.

✓

Use teams for groups

Where a team shares the same responsibility, assign roles to the team and manage membership over time.

✓

Review after role changes

Ask users to refresh their session and confirm they can see the correct app areas, views, queues, and dashboards.

✓

Remove access for leavers

Deactivate users or remove roles promptly when people leave the support function.

Troubleshooting

Access problems and checks

01

User cannot see Support

Confirm the user has a Support role and can access the model-driven app.

02

User cannot see a queue

Confirm queue membership, ownership, team access, and security role assignment.

03

User cannot maintain Managed Services

Confirm they have Support Account Manager or Support Admin access.
