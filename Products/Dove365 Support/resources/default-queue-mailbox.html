Default Queue and Mailbox Intake

Configure how new support work enters Dove365 Support from email, queues, and intake automation.

Intake Model

How support work enters the system

Dove365 Support separates the Case from the Queue Item. The Case stores the support issue. The Queue Item controls which queue owns the work and who is actively working it.

01

Email or manual Case creation

Cases can be created manually by a user or automatically from the support mailbox triage flow.

02

Default queue placement

New email-created Cases are added to the configured default support queue as Queue Items.

03

Pickup and ownership

When a user picks work, the Queue Item Worked By value is set and the item appears in that user’s work lists.

Configure Default Queue

Run Configure Default Queue

  1. Create or identify the default intake queue.
  2. Copy the queue GUID.
  3. Set dove365_DefaultQueueGUID.
  4. Run Configure Default Queue manually after import or after changing the queue.
  5. Confirm the flow completes successfully.
If the default queue is wrong, email-created Cases and available-to-pick dashboard counts will be wrong.
Mailbox Triage

Validate support mailbox automation

✓

Mailbox address

Confirm the monitored mailbox is the mailbox customers use for support requests.

✓

Connection references

Confirm Outlook and Dataverse connections are authenticated and owned by the intended service account or administrator.

✓

Flow run history

Review successful and failed runs after testing. Failures usually indicate mailbox, connection, permission, or queue configuration issues.

✓

Case and Queue Item records

Confirm the email creates a Dove365 Support Case and a Queue Item in the default queue.

Operational Checks

What admins should confirm

Area Expected result
Default queue The configured queue exists, is active, and is accessible to support users or teams.
Email triage New support emails create Cases and Queue Items without manual intervention.
Worked By Picked queue work is assigned to the user actively working the item.
Dashboards Available-to-pick and open-case counts reflect the configured queue and active support records.
