Documentation / SharePoint Integration

SharePoint Integration Guide

Store and access documents against your CRM records. Understand how SharePoint document management works inside Dove365 CRM — Starter and the business benefits it delivers.

The Document Problem

Customer files where you need them

Without integrated document management, customer files live in email attachments, personal drives, and shared folders with no clear connection to the customer record. Dove365 CRM — Starter’s SharePoint integration solves this directly.

01

Files linked to CRM records

Documents stored through Dove365 CRM — Starter are held in SharePoint and linked directly to the relevant account, contact, opportunity, or case. Open the record and the files are there.

02

No duplicate storage

Files live in SharePoint — your existing Microsoft 365 document platform. CRM surfaces them in context. No synchronisation required, no separate document system to manage.

03

Access control via Microsoft 365

File permissions are managed through SharePoint and Microsoft 365, using the same access controls your organisation already has. No separate document permission system to configure.

How It Works

SharePoint folders, linked to CRM

When SharePoint integration is enabled, Dove365 CRM — Starter links SharePoint document folders to your CRM records. Files uploaded from within a CRM record are stored in SharePoint and remain permanently linked.

Document access is supported for Contacts when the Contact table is selected in Document Management Settings, in addition to the other configured CRM record types.
1

Record exists in CRM

An account, contact, opportunity, or case is created in CRM. A corresponding SharePoint document folder is linked to that record.

2

Files uploaded in context

Users upload files from within the CRM record — no need to navigate to SharePoint manually. The file appears in both CRM and SharePoint immediately.

3

Collaborate in SharePoint

The underlying SharePoint document library supports co-authoring, version history, and standard Microsoft 365 collaboration features.

4

Access from CRM or SharePoint

Files can be accessed from within the CRM record or directly in SharePoint, depending on the user’s preference and workflow.

Business Benefits

What SharePoint integration delivers

The integration between Dove365 CRM — Starter and SharePoint removes the document management problem that most small businesses face — files scattered across inboxes, drives, and chat tools with no connection to the customer.

✓

Complete customer records

A CRM account isn’t just contact details. With SharePoint integration, it contains proposals, contracts, reports, and supporting documents — all in one place, against the right record.

✓

Faster information retrieval

Find a customer’s files by opening their CRM record — not by searching email threads or asking colleagues where the document was saved last time.

✓

Version history

SharePoint maintains a version history for every document, so you can see how a proposal or contract evolved and restore previous versions when needed.

✓

Collaboration without forwarding

Multiple team members can work on the same document in SharePoint without emailing versions back and forth. The CRM record always shows the most current version.

✓

Governance and audit trail

Files stored in SharePoint via CRM are governed by your organisation’s SharePoint permissions and retention policies, providing a consistent and auditable document trail.

✓

Contact-level document storage

SharePoint integration can be enabled for contacts specifically, so each contact record has its own associated document folder — useful for client-specific files and compliance-driven document management.

Security and Control

Your data stays under your control

SharePoint integration with Dove365 CRM — Starter uses your organisation’s existing Microsoft 365 infrastructure. Dove365 does not store your documents. Everything stays in your Microsoft 365 tenant.

Permissions managed in Microsoft 365 File access is controlled through SharePoint and Entra ID — not the CRM. Your existing Microsoft 365 permission structure applies to all documents accessed through Dove365 CRM — Starter.
Data stays in your tenant All documents are stored in your organisation’s SharePoint. There is no third-party storage involved. You retain full ownership and complete control over all files.
CRM access governed by security roles Users must have the appropriate CRM security role to access the Documents tab on a record. Dove365 CRM — Starter’s role-based access controls what each user can see and interact with.
SharePoint integration is configured by your CRM administrator as part of the Dove365 deployment. If documents are not appearing on CRM records, or if you cannot upload files from within CRM, contact your admin to verify the integration settings are correctly configured for that record type.
